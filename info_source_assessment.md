# 新信息源 / 建模方式 评估报告

> 评估对象：山东省日前(D+1)直调负荷预测系统 **v6**（官方验证 MAE = 1445.62 MW，R² = 0.9292）
> 评估目标：系统性判断是否存在新的可落地信息源或建模方式可降低午间误差，**是否值得进入实验阶段**。
> 评估方法：数据可得性核查 + 三个 walk-forward 跨年探针（LightGBM，与生产同款）+ 评价体系完善。
> 产物：`eval_system.py`（评价体系）、`exp_info_probe.py`（误差反馈+辐照变化探针）、
> `exp_midday_model_probe.py`（午间专项模型探针）；结果 CSV 各一份；本报告。
>
> **合规声明**：未修改 train.py / predict.py / features.py / 生产 bundle / v6 归档。所有探针为独立诊断脚本，
> 真实负荷仅作评估目标与（严格 lag≥2 天的）已实现历史反馈特征。任何候选未通过跨窗口稳定性验证前不进入生产。

---

## 总判断（TL;DR）

- **可立即实验的方向：无。** 三个可在不新增外部数据下测试的方向（历史误差反馈、辐照变化、午间专项模型）
  经 walk-forward 跨年探针全部**未通过迁移性检验，且在官方验证窗均使误差增大**。
- **可迁移性失败的根因（数据证实）**：用现有特征预测外部预测误差 `ext_error`，baseline LightGBM 在三内折
  中 R² = **−0.72 / +0.09 / −3.80**（2/3 为负），即现有特征**无法跨年稳定预测外部误差**；外部误差以
  跨年漂移为主，非可学习信号。
- **结论**：当前 v6 已逼近现有信息空间上限。下一阶段唯一具突破潜力的方向是**引入新的外部可观测数据**
  （新能源出力、云量、分区域 NWP、外部误差实时反馈），而非继续挖掘旧特征或调整模型结构。

## A. 当前系统剩余误差来源分析

基于 `eval_system/`（分时/分天气）与 `FDS_midday/`（午间专项）诊断：

| 来源 | 量化证据 | 是否可从现有信息消除 |
|---|---|---|
| **外部预测负荷误差** | 午间外部 MAE 3449，模型仅降至 2835（改善 18%）；**82% 不可消**；Top19/20 失败点集中在 2 个外部崩盘日（2026-04-13 +13000、2026-03-03 −13000）；baseline 跨年 R² 多为负 | **否**（外部预测自身误差，模型共享同一 pred_load 输入，无信号） |
| **光伏/新能源波动不可观测** | 多云午间(clearness 0.2-0.5) MAE 3818、std 4827 最高；残差非白噪声但现有特征 R²=−0.03 | **否**（数据中无新能源出力，仅有省级辐照） |
| **高负荷外部漏峰** | actual P90+ Bias −4316；P90-95 pred_load ext_Bias −3485（外部漏峰） | **否**（=ext_error，不可从 pred_load 定位） |
| **温度极端偏置** | 高温(≥30°C) Bias **+1347**（高估）；低温(<8°C) Bias −539（低估） | 部分（threshold_corr temp<8 已校；高温未覆盖，但跨年漂移） |
| **11:00 爬坡定时低估** | slot 44-47 Bias −850~−1420，3 月最差 −1153 | 跨年符号翻转，不可迁移（见下） |
| **跨年概念漂移** | 同季节 2025 春 model_Bias +1971 → 2026 春 −572（**符号翻转** 2543 MW） | 否（漂移本身不可学） |

**核心**：剩余误差的**主体是"不可观测 / 不可迁移"成分**，而非模型容量或特征工程不足。这与既有的
CS1/CS2/CS3 失败、1300 目标不可行、技术评估 4 方向放弃等结论一致：v6 近信息下限。

## B. 候选改进方向列表

每个方向标注：是否需新数据 / 是否违反泄露约束 / 工程难度 / 预期收益 / 验证方法。**已探针测的方向附实测结果。**

### B1. [FB] 历史预测误差反馈（Part 4）— ❌ 已验证不值得

- 需新数据？否（用既有 actual + pred_load 构造过去误差）。
- 泄露约束？**需严格 lag≥2 天**。D+1 部署 09:00 day D 预测 day D+1：运行日 day(T)-1 全天 actual
  09:00 尚不可得；仅 day(T)-2 及更早全天 actual 已实现。故 `error_lag_1day`（紧前日）**在部署时刻不可得**，
  须用 lag≥2d。本探针已按此实现（fb_lag2d/fb_lag7d/fb_mean7d 均基于 T-2d 及更早）。**不违反泄露**。
- 工程难度：低（特征拼接）。
- 预期收益：**实测为负**。walk-forward（baseline=现有特征，目标=ext_error）：
  - 三内折 ΔMAE = [−170.7, +25.0, −145.4]（**不一致**，秋25 反增）；
  - 官方 val ΔMAE = **+106.1 MW**（午间 +205），R² 由 +0.135 降至 +0.055。
  - ext_error 日均自相关 ACF(1)=0.294 有弱持续性，但**不跨年迁移**（2025 误差模式误导 2026）。
- 验证方法：已用 4 折 walk-forward LightGBM；结论**不通过**。
- **判定：不值得进入实验阶段。**

### B2. [IC] 辐照变化特征（Part 2，irrad_change_15min/1h/std）— ❌ 已验证不值得

- 需新数据？否（从现有 `irrad` 派生 ic_d15/ic_d1h/ic_std96/ic_d1d）。
- 泄露约束？无（预报辐照，D+1 已知）。
- 工程难度：低。
- 预期收益：**实测为负**。三内折 ΔMAE = [−28.0, −19.0, +29.6]（**不一致**，冬26 反增 +30、午间 +300）；
  官方 val ΔMAE = **+81.5 MW**（午间 +146）。
- 验证方法：已 4 折 walk-forward；**不通过**。
- **判定：不值得进入实验阶段。**

### B3. [MID] 午间专项模型（Part 3 方案A，午间/非午间拆分）— ❌ 已验证不值得

- 需新数据？否（同一批现有特征）。
- 泄露约束？无。
- 工程难度：中（双模型管线、校正重估）。
- 预期收益：**实测不可迁移**。午间专用 vs 全天模型，午间 test ext_error MAE：
  - 三内折 Δ = [−1191.2, +142.9, −191.2]（**不一致**，秋25 反增）；
  - 官方 val Δ = **+149.1 MW**（更差）。
  - 春25 大幅"改善"(−1191) 正是过拟合信号：2025 春午间 Bias 极端(+1971)，专用模型捕获但**不泛化**。
- 验证方法：已 4 折 walk-forward；**不通过**。
- **判定：不值得进入实验阶段。**（全天不会退化，但午间无可迁移收益）

### B4. [LR] level + ramp 模型（Part 3 方案B）- ❌ 已实验否决

- 需新数据？否。泄露安全 operationalization：anchor(t)=actual(t-2d)（已实现，lag≥2d 合规），ramp target r(t)=actual(t)−anchor，重构 actual_hat=anchor+r_hat。
- 泄露约束？anchor 用 past actual（lag≥2d，安全），但**需放宽 invariant #1**（actual 入特征）。
- 工程难度：高（递归预测 + 校正重构）。
- 预期收益：**实测大幅为负**。walk-forward 4 折（exp_level_ramp_probe.py，目标=actual）：
  - V3 level+ramp vs V1 baseline：全天 ΔMAE = [**+1860, +3270, +2135**]，午间 ΔMAE = [**+3622, +6809, +6393**]；官方 val ΔMAE = **+2143**（午间 +5037）。**三折+val 全部大幅恶化**——ramp(2日变化)目标远比 level 难预测，重构放大误差。
  - 附带发现 V2(baseline + actual(t-2d) anchor)：内折 ΔMAE=[−159,−296,−427]（改善），但**官方 val 仅 −27 MW（噪声级，不可迁移）**；且需放宽 invariant #1。即便放宽，val 收益亦不显著。
- 验证方法：已 4 折 walk-forward；**不通过（大幅恶化）**。
- **判定：实验否决，不值得进入实验阶段。**

### B5. [CLOUD] 云量信息（Part 2，cloud_cover/low/mid/high）— ⏳ 须等待新数据

- 需新数据？**是**。当前气象 CSV 仅 5 基础变量（风电风速/光伏温度/降水/风速/辐照度）×{raw,p25,p50,p75,std}，
  **无任何云量字段**。
- 泄露约束？无（NWP 云量预报）。
- 工程难度：中（需 NWP 云量字段接入）。
- 预期收益：中。多云午间方差最高(std 4827)，云量分层(low/mid/high)可能解释部分光伏波动；但光伏出力更直接。
- 验证方法：取得云量数据后，构造特征 + 4 折 walk-forward LightGBM 增量 R²/MAE。
- **判定：等待新数据后再评估。**

### B6. [REGION] 分区域气象（Part 2，鲁东/鲁西/鲁南/鲁北）— ⏳ 须等待新数据

- 需新数据？**是**。当前为省级聚合 + 空间分位(p25/p50/p75/std 已部分捕获网格扩散)，**无显式四区域拆分**。
- 泄露约束？无。
- 工程难度：中高（区域 NWP 切片 + 对齐）。
- 预期收益：低-中。空间分位已含部分信息；显式区域拆分边际增量有限（山东负荷/光伏分布相对集中）。
- 验证方法：取得区域数据后 walk-forward 增量评估。
- **判定：等待新数据，优先级低于 B5/B7。**

### B7. [RENEW] 新能源出力（光伏/风电实际/预测出力）— ⏳ 须等待新数据，**潜力最高**

- 需新数据？**是**。当前 data/ 仅有负荷与气象，**无任何新能源出力**。
- 泄露约束？使用**已实现历史出力**(lag≥2d)或**日前新能源预测**均无泄露；禁用预测日之后的实际出力。
- 工程难度：中（需接入新能源出力序列，与负荷时间轴对齐）。
- 预期收益：**高（唯一可能突破 2835 下限的方向）**。午间误差主体=光伏波动+外部漏峰，直接观测/预测新能源
  出力可补足"不可观测"成分；高温高估(+1347)、寒潮低估亦可能与新能源/供暖出力相关。
- 验证方法：取得数据后构造增量特征（出力及其 lag/异常），4 折 walk-forward 跨年验证增量 MAE；严格泄露检查。
- **判定：等待新数据，**最高优先级**。**

### B8. [EXTFB-RT] 外部预测误差实时反馈（FB 的实时增强版）— ⏳ 须等待新数据

- 需新数据？**是**。需部署端提供"外部预测在近期的实时误差反馈"管线（非仅历史 actual 回算）。
- 泄露约束？实时已实现误差可用；禁用未来 actual。
- 工程难度：高（需外部预测误差的实时回传管线）。
- 预期收益：中。B1 历史 FB 不迁移，但若外部预测误差存在**当日内可观测的实时偏置**（如晨间实际 vs 预报），
  或可部分修正；不确定性高。
- 验证方法：取得实时反馈流后 walk-forward。
- **判定：等待新数据，优先级中。**

### B9. [NWP-BETTER] 更准/更高频 NWP — ⏳ 须等待新数据

- 需新数据？**是**（升级 NWP 源或起报频次，如晨间 issue）。
- 泄露约束？无。
- 工程难度：高（数据源替换/对接）。
- 预期收益：中。9AM 部署依赖晨间 issue（已有晨间 issue 代理）；更准 NWP 直接降气象驱动误差。
- 验证方法：替换 NWP 后全管线 walk-forward MAE。
- **判定：等待新数据，优先级中。**

## D. 明确判定

### 值得实验（现在）
**无。** 在不新增外部数据的前提下，所有可测方向（B1/B2/B3/B4）均已被 walk-forward 跨年探针否决。
现有特征对 ext_error 的跨年预测 R² 多为负，证明**剩余误差非可学习信号**。

### 属于继续过拟合（应停止）
- **B3 午间专项模型、B4 level+ramp**：同一特征空间内的结构重组，无新信息；B3 实测 val +149 MW、B4 实测 val +2143 MW（均恶化）。
- **B1 历史误差反馈、B2 辐照变化**：看似"新信息"实为旧特征的派生/已实现历史的回算，跨年不迁移，val 分别 +106/+82 MW。
- 任何在现有特征上继续调 threshold_corr/drift_corr/hour_bias/分位阈值 的方向（CS1/CS2/CS3 已证不可迁移）。
- 任何"单窗口 val 改善即采纳"的做法（违反跨窗口稳定性原则）。

### 必须等待新数据
- **B7 新能源出力（最高优先）**：直接补足"不可观测的光伏波动 + 外部漏峰"，是唯一可能突破信息下限的方向。
- **B5 云量、B6 分区域、B8 外部误差实时反馈、B9 更准 NWP**：均需新外部数据源，取得后须做泄露检查 + walk-forward。

---

## 最终原则

当前 v6 已证明**模型结构不是主要瓶颈**：用生产同款 LightGBM 在现有特征上预测外部误差，跨年 R² 多为负，
午间专用模型/误差反馈/辐照变化均不可迁移。下一阶段重点应为**寻找新的可观测信息**（首选新能源出力，
其次云量/分区域 NWP/外部误差实时反馈），而非继续挖掘已被模型充分利用的旧特征。

**落地建议**：暂停模型侧实验；推动数据侧接入新能源出力（B7）与云量（B5）数据源；取得后按
`exp_info_probe.py` 的 walk-forward 跨年范式（4 折、lag≥2d 泄露控制、官方 val 仅作参考不作迁移依据）
评估增量收益，确认跨折一致降低 MAE 后方可考虑进入生产实验。
