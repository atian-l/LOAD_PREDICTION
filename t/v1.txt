# Goal
Objective（最终目标）
构建一个山东省全省日前（D+1）直调负荷预测系统，使用 Python 开发，并支持训练模式与预测模式两种运行方式。
训练模式用于模型开发、训练和评估，能够生成完整的逐点预测结果、逐点误差统计以及验证集评估结果；预测模式用于项目部署后的日常运行，每天仅利用运行时能够获得的数据，对 D+1 全天（96 个 15 分钟时刻）进行直调负荷预测，重新训练模型的周期可自行选择。

# Inviolable Constraints

## 1. 数据隔离（最高优先级）
direct_load_data.csv 中的：
实际直调负荷

只能作为最终评估基准。

禁止用于：
- 模型输入；
- 特征工程；
- 滞后特征；
- 滚动窗口；
- 统计特征；
- 数据填补；
- 标准化或归一化计算；
- 编码；
- 目标变换；
- 任意中间变量；
- 任何形式的训练过程。

整个预测流程中，模型不得直接或间接接触任何真实负荷数据。

## 2. 外部预测数据使用约束
direct_load_data.csv 中的：
预测直调负荷

属于外部预测数据。

允许：
- 作为输入特征；
- 构造滞后特征。

必须满足：
- 运行当天最远只能获得 D+1 数据；
- 不得假设能够获得 D+2 或之后的预测值。

## 3. 滞后特征约束
滞后特征：
- 只能基于预测直调负荷构造；
- 严禁使用任何真实负荷构造滞后项；
- 最短滞后必须包含：
lag_192

允许增加更多更长滞后。

## 4. 气象数据约束
气象数据：
shandong_weather_15min.csv

对于相同预测时间：
必须仅保留起报时间最晚的记录（丢弃较早起报时间的版本）。

例如：
预测时间相同：2025/01/25 08:00
若存在起报时间：
2025/01/24 08:00
2025/01/24 20:00
必须选择：2025/01/24 20:00

该规则在训练、验证和预测阶段均必须严格执行。

## 5. 时间边界约束
训练阶段不得使用验证集时间范围（≥2026/02/01 00:00:00）的数据。
预测阶段不得使用：
- 未来真实负荷；
- 未来气象实况；
- 超出运行时可获得范围的外部预测数据。

所有输入必须符合真实生产环境的数据可获取性。

## 6. 工程约束
代码必须位于：
\load_prediction\load_pred

输出必须位于：
\load_prediction\output

模型必须保存于：
\load_prediction\models

所有输入数据必须读取自：
\load_prediction\data

整个项目必须使用 Python 实现。

如需新增气象变量，可使用以下 CDS API（请自行替换密钥）（严防数据泄露）：
url: https://cds.climate.copernicus.eu/api
key: cc121503-9391-48e5-886b-3c78c27a0f9d

训练模式与预测模式必须完全解耦，预测模式仅加载训练完成后的模型进行推理，不得包含任何重新训练逻辑。

## 7. Agent Loop 自主迭代
系统可采用自主迭代闭环：
1. 执行训练并在验证集上评估 MAE；
2. 若 MAE ≥ 1500 MW，自动分析可能原因（如特征不足、模型欠拟合、超参数不合理等）；
3. 调整策略（重新构造特征、调整模型结构或超参数等），再次训练并评估；
4. 重复上述过程，直至验证集 MAE 稳定低于 1500 MW；
5. 保存最终模型至 \load_prediction\models。

循环过程中任何时候均不得违反上述数据隔离、时间边界等约束。
